## 📚 엄격 모드

자바스크립트는 오랫동안 기존의 기능을 변경하지 않고 새로운 기능을 추가하며 호환성 이슈 없이 발전했습니다. 하지만 이는 곧 기존 기능에서의 실수나 불완전한 부분까지 가져가게 된다는 단점이기도 했습니다.<br/>
이후 ECMAScript5(ES5)가 등장하면서 새로운 기능 추가와 동시에 기존 기능 중 일부가 변경되었습니다. 기존 기능을 변경하였기 때문에 하위 호환성 문제가 생길 수 있게 되었습니다. 이를 해결하기위해 기본 모드에선 변경사항 대부분이 활성화되지 않고, `'use strict';`라는 특별한 지시자를 사용했을 때에만 활성화되도록 하였습니다.

### 'use strict';

스크립트의 최상단에 오면 스크립트 전체가 `모던한` 방식으로 동작하게 됩니다.

```js
'use strict';

```

아래와 같은 방식으로 함수 내에서만 엄격 모드를 사용할 수도 있습니다.

```js
function name() {
  'use strict';
}
```

> 주석을 제외하곤 다른 어떤 코드보다 최상단에 위치해야합니다.<br/>
> 엄격 모드를 실행하고 취소할 수 있는 코드는 없습니다.

<br/>

<strong>엄격 모드는 반드시 사용해야할까요?</strong><br/>

> 'use strict';를 하지 않아도 `클래스`와 `모듈`을 사용한다면 자동으로 엄격 모드가 적용됩니다.<br/>
> 일반적으로 엄격 모드를 사용하는 것이 예상치 못한 에러를 방지할 수 있으므로 사용하는 게 좋습니다. 🤪
